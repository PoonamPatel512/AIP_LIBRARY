# Architecture

## Purpose

This folder stores architecture documentation that explains how the system is structured and why those boundaries exist.

## What Belongs Here

- Architecture overviews
- Layer and boundary documentation
- System structure notes
- Architecture summaries that guide implementation

## Naming Conventions

- Use names that clearly reflect the system or subsystem being described.
- Prefer topic names over session-based labels.

## Rules

- Keep architecture guidance separate from decisions and raw source material.
- Do not mix implementation code explanations with product requirements.
- Keep documents current with the repository’s actual structure.
