# API

## Purpose

This folder stores API design knowledge and contract references for server-facing interfaces.

## What Belongs Here

- API contract summaries
- Route design notes
- Request and response conventions
- Integration-facing documentation

## Naming Conventions

- Use route, resource, or capability names.
- Keep names consistent with the public contract.

## Rules

- Do not store implementation code here.
- Keep API guidance separate from UI and database notes.
- Preserve contract clarity and stability.
