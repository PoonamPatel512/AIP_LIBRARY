# 00 Overview

## Purpose

This folder is the entry point for the documentation system. It should orient contributors to the repository’s product, architecture, and knowledge structure.

## What Belongs Here

- High-level documentation maps
- Documentation navigation guides
- Cross-references to source, knowledge, architecture, and decision records

## Naming Conventions

- Use numeric prefixes only when they help establish reading order.
- Use short, explicit names for files that define the documentation system.

## Rules

- Keep this folder high-level and navigational.
- Do not place implementation notes or working drafts here.
- Keep content stable and easy to scan.
