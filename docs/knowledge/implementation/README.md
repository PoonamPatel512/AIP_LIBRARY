# Implementation Knowledge

## Purpose

This folder stores implementation-focused knowledge that can guide future development without becoming code.

## What Belongs Here

- Implementation summaries
- Technical patterns
- Reusable engineering notes
- Known constraints and conventions

## Naming Conventions

- Use architecture or subsystem names.
- Keep names stable and meaningful to engineers.

## Rules

- Do not store source documents here.
- Do not replace ADRs or architecture docs with informal notes.
- Keep knowledge aligned with the repository’s standards.
