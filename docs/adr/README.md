# ADR

## Purpose

This folder stores architecture decision records that capture durable tradeoffs and resolved design choices.

## What Belongs Here

- ADRs
- Formalized architecture decisions
- Significant product or technical tradeoffs that should survive beyond a single work session

## Naming Conventions

- Use stable ADR identifiers.
- Prefer titles that state the decision clearly.

## Rules

- Keep each ADR focused on one decision.
- Do not store brainstorming notes here.
- Do not rewrite old ADRs unless a decision is being formally superseded.
