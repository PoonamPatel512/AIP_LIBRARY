# Database

## Purpose

This folder stores database design knowledge, schema guidance, and persistence architecture notes.

## What Belongs Here

- Database design documentation
- Schema guidance
- Migration notes
- Persistence model summaries

## Naming Conventions

- Use names that reflect the subject table, model, or persistence topic.
- Keep terminology aligned with the Prisma and PostgreSQL model.

## Rules

- Do not store raw data exports here.
- Keep persistence reasoning separate from application code.
- Preserve schema intent and provenance concerns.
