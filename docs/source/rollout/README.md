# Rollout Source

## Purpose

This folder stores original rollout plans, launch notes, and implementation rollout artifacts.

## What Belongs Here

- Rollout plans
- Launch checklists
- Migration and release source notes
- Operational rollout references

## Naming Conventions

- Use dates or release identifiers when they are part of the source artifact.
- Prefer names that match the original rollout material.

## Rules

- Keep source rollout material separate from meeting notes and decisions.
- Do not add implementation commentary here.
- Preserve the operational intent of the source document.
