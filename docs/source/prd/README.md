# PRD Source

## Purpose

This folder stores original product requirements documents and related business source material.

## What Belongs Here

- Original PRDs
- Business requirement exports
- Requirement artifacts that predate or support implementation planning

## Naming Conventions

- Preserve the source document’s identity in the file name.
- Use date or version markers only when they exist in the original artifact.

## Rules

- Do not summarize or reinterpret requirements here.
- Keep authoritative business wording intact.
- Store only source material, not architectural responses.
